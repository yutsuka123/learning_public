# cpp_m/README.md

[重要] `cpp_m/` は「モダンC++の学習」を目的に、**標準バージョン別に分けたサンプル**を置くフォルダです。  
[推奨] まず `main.cpp` を読み、次に `cpp11.cpp` → `cpp14.cpp` → `cpp17.cpp` → `cpp20.cpp` → `cpp23.cpp` → `modern.cpp` の順で進めます。理由: 機能追加の流れが自然で、理解が積み上がるため。  

## ファイル構成
- `main.cpp`
  - このフォルダの入口。学習の進め方と、コンパイル時の標準（`__cplusplus` / `_MSVC_LANG`）の確認を行います。
- `cpp11.cpp`
  - C++11: `auto` / 範囲for / `nullptr` / `enum class` / `unique_ptr` / ラムダ など
- `cpp14.cpp`
  - C++14: ジェネリックラムダ / `make_unique` / 返り値型推論 など
- `cpp17.cpp`
  - C++17: 構造化束縛 / `if constexpr` / `optional` / `string_view` など
- `cpp20.cpp`
  - C++20: `concepts` / `span` / `constinit` など
- `cpp23.cpp`
  - C++23: **機能テストマクロ**を使って、利用可能なら C++23 の新機能を実演（未対応環境でもコンパイルできる形）
- `modern.cpp`
  - 最新（原則 C++23）を想定した「総合」。小さな問題を題材に、読みやすい書き方・例外/エラー表示・標準機能の組み合わせを練習します。

## ビルド/実行
ビルド手順は `INSTALL.md` を参照してください（[厳守] 変更時は更新）。  

## このフォルダの設計方針（学習用）
- **可読性重視**: 1つの機能を詰め込みすぎず、説明用の関数に分けます。
- **エラー表示を丁寧に**: 例外時は「どの関数で、どの引数で」起きたかを出します。
- **[禁止] 省略しすぎる書き方**: 学習の妨げになるため（ただし `modern.cpp` では、読みやすさの範囲で簡潔さも意識します）。
