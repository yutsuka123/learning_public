# IoT エラー仕様書

[重要] 本書は Cloud / Local / ESP32 で共通利用するエラー体系を定義する。  
理由: ログ監視・問い合わせ・再現試験で同一コードを使い、切り分けを高速化するため。

## 1. 基本方針
- [厳守] エラーコードは `000`〜`999` の3桁固定で管理する。
- [厳守] エラーなしは `000`（`ERR000`）とする。
- [厳守] ログ出力は `ERRxxx` 形式で出し、`ERR` 検索で抽出可能にする。
- [禁止] `E12` / `ERR-12` など桁数が揺れる表記を使わない。

## 2. 番号帯ルール
- `001`〜`099`: 起動時・初期化エラー（ESP32専用）
- `100`〜`199`: ネットワーク系（MQTT / HTTP / HTTPS / OTA）
- `200`〜`299`: 認証・セキュリティ系
- `300`〜`399`: 内部通信エラー
- `400`〜`499`: その他
- `500`〜`599`: ローカル特有
- `600`〜`699`: サーバー特有
- `700`〜`799`: クラウド系
- `800`〜`899`: 外部機器・ハード系
- `900`〜`999`: 致命的・アボート系

## 3. 現在の定義（抜粋）
- `ERR000`: エラーなし
- `ERR001`: ESP起動時 I2C タスク開始失敗
- `ERR002`: Wi-Fi初期化要求送信失敗
- `ERR003`: Wi-Fi初期化タイムアウト
- `ERR004`: MQTT初期化要求送信失敗
- `ERR005`: MQTT初期化タイムアウト
- `ERR006`: TimeServer初期化要求送信失敗
- `ERR007`: TimeServer初期化タイムアウト
- `ERR101`: Wi-Fiタスクエラー
- `ERR110`: MQTTタスクエラー
- `ERR120`: HTTPタスクエラー
- `ERR130`: OTAタスクエラー
- `ERR131`: TimeServerタスクエラー
- `ERR300`: 内部通信エラー
- `ERR800`: 外部機器・ハード系エラー
- `ERR900`: 致命的エラー（一般）
- `ERR901`: mainTask起動失敗
- `ERR999`: 予期しない致命エラー

## 4. 実装位置
- 共通定義: `shared/include/error.h`
- ESP32利用例: `ESP32/src/main.cpp`
