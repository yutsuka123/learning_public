# IoT 安全消去ポリシー

[重要] 本書はアンインストール・廃止時に機密情報・認証情報を安全に消去するポリシーを定義する。  
理由: 「アンインストール=通常削除のみ」で機密残留を許容しないため。003-0018 で文書化。

[厳守] 「何を、どこまで、どう消したか」を再現可能な手順として定義する。  
理由: 監査・再現試験・インストーラ実装の参照先とするため。

[禁止] 「アンインストール=通常削除のみ」で機密残留を許容する運用。  
理由: 機密ファイルの論理削除だけでは復元ツールで読み出し可能なため。

## 1. 対象モジュール別消去対象
### 1.1 LocalServer
| 対象 | 場所 | 消去方法 | 備考 |
|------|------|----------|------|
| 永続設定 | `data/settings.json` | 上書き消去 | 機密値が含まれる場合あり |
| 鍵ストア | `data/keyStore.json` | 上書き消去 | 必須 |
| セキュリティ状態 | `data/securityState.json` | 上書き消去 | 必須 |
| 監査ログ | `logs/*.log` | 上書き消去 | 認証失敗等の記録を含む |
| アップロードキャッシュ | `uploads/*` | 上書き消去 | 一時ファイル |
| OTA配布物 | `ota/firmware.bin` 等 | 通常削除可 | メーカー署名済み配布物、流出時は再配布可能 |
| 証明書・秘密鍵 | `certs/*` | 上書き消去 | 必須。LocalServer専用のもの |

### 1.2 ProductionTool（工場端末）
| 対象 | 場所 | 消去方法 | 備考 |
|------|------|----------|------|
| 鍵素材 | 設定で指定した領域 | 上書き消去 | 製造用秘密鍵は絶対に残留禁止 |
| 実行履歴 | 製造ログ保存先 | 上書き消去 | 機密が含まれる場合あり |
| 一時ファイル | 作業用temp領域 | 上書き消去 | 中間生成物 |

### 1.3 LocalSoft / LocalSoftForCloud（廃止時）
| 対象 | 場所 | 消去方法 | 備考 |
|------|------|----------|------|
| SQLite/ローカルDB | DBファイル | 上書き消去 | 認証情報・設定を含む |
| キャッシュ | キャッシュディレクトリ | 上書き消去 | |
| 機密設定 | 設定ファイル | 上書き消去 | |

## 2. 消去方法の定義
### 2.1 通常削除
- OSのファイル削除API（`unlink` 等）。論理削除のみ。
- [禁止] 機密対象には通常削除のみを採用しない。

### 2.2 上書き消去
- 削除前にファイル内容を上書きする。
- [推奨] 上書きパターン: 0x00、0xFF、乱数の3回、または1回以上。
- [重要] Windows: `FILE_FLAG_WRITE_THROUGH` または同等で確実にメディアへ反映する。
- [将来対応] macOS: `srm` または `SecureEmptyTrash` 相当。Linux: `shred` 相当。

### 2.3 OS安全削除API
- [推奨] Windows: `CryptGenRandom` で上書きバッファを生成し、`WriteFile` で上書き。または `SecureDelete` 相当のAPIを使用。
- [推奨] 実装時はプラットフォームの推奨方式を採用する。

## 3. 消去成否の監査ログ
- [厳守] アンインストール実行時、以下を監査ログへ記録する。
  - 消去対象一覧（パス、種別）
  - 各対象の消去結果（成功/失敗、失敗理由）
  - 実行日時
  - 実行主体（ユーザー/インストーラ）
- [重要] 監査ログは消去対象外の領域（例: インストーラ専用ログディレクトリ）へ保存し、アンインストール完了後も残す。
- [推奨] 監査ログ形式は JSON または構造化テキストとし、後から検索・集計可能にする。

## 4. 再現可能な手順
### 4.1 LocalServer アンインストール手順
1. LocalServer プロセスを停止する。
2. `data/keyStore.json`、`data/securityState.json`、`data/settings.json` を上書き消去する。
3. `logs/*.log` を上書き消去する。
4. `uploads/*` を上書き消去する。
5. `certs/` 配下の LocalServer 専用証明書・秘密鍵を上書き消去する。
6. 上記以外の `data/`、`logs/`、`uploads/` を通常削除する。
7. 消去成否を監査ログへ記録する。

### 4.2 ProductionTool アンインストーラ手順
1. ProductionTool プロセスを停止する。
2. 鍵素材保存領域を上書き消去する。
3. 製造ログ・一時ファイルを上書き消去する。
4. 消去成否を監査ログへ記録する。

### 4.3 LocalSoft 廃止時移行・消去手順
1. LocalServer 等への移行が完了していることを確認する。
2. LocalSoft プロセスを停止する。
3. SQLite/ローカルDB を上書き消去する。
4. キャッシュ・機密設定を上書き消去する。
5. LocalSoftForCloud 旧資産も同等の消去ポリシーで回収する。
6. 消去成否を監査ログへ記録する。

## 5. 試験仕様への反映
- [厳守] アンインストール試験では以下を確認する。
  - 消去対象が定義どおり消去されること。
  - 上書き消去対象が通常削除のみでないこと（実装が上書きを行っていること）。
  - 監査ログに消去対象・結果・日時が記録されていること。
- 試験番号: 7090（安全消去試験）を `試験仕様書.md` で追加する。
- [重要] 7090 試験前に k-user バックアップ用ファイル（keyStore.json）をインストールルート外へ退避すること。理由: 試験後の k-user 再発行を不要にするため。手順は `コマンド仕様書.md` 4.1.1 を参照。
