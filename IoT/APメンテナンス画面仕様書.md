# APメンテナンス画面仕様書

[重要] 本書は ESP32 の AP メンテナンスモード（`http://192.168.4.1/`）の画面仕様、認証ロール、起動条件を定義する。  
理由: 現地復旧と製造保守で同じUIを使うため、権限制御と表示項目を先に固定して実装ブレを防ぐため。

[厳守] APモード中は MQTT タスクを起動しない。通常モード専用タスク（クラウド接続前提タスク）も起動しない。  
理由: APモードは局所メンテナンス専用であり、通常運用経路と同時起動すると誤操作とリソース競合を招くため。

[厳守] APモード ⇔ 通常モードの切替は、原則として再起動を伴う。  
理由: タスク起動構成、ネットワークスタック構成、認証状態を確実に初期化するため。

## 1. APモード遷移条件
- [厳守] 起動時ボタン3秒長押しで AP モードへ入る。
- [厳守] MQTT `call/maintenance` 指令受信時は、次回起動を AP モードにして再起動する。
- [厳守] Wi-Fi / MQTT の必須設定未投入（SSIDや broker 情報欠落）時は、自動で AP モード起動する。

## 2. APモード表示仕様
- [厳守] 電源LED（青）パターン:
  - 0.3s ON -> 0.3s OFF -> 0.3s ON -> 0.3s OFF -> 1.0s ON -> 0.3s OFF を繰り返す。
- [厳守] LCD 1行目に `MaintenanceMode` を表示する。
- [推奨] LCD 2行目に AP 名（16文字以内）を表示する。

## 3. APネットワーク仕様
- AP URL: `http://192.168.4.1/`
- SSID: `AP-esp32lab-<MAC(no colon)>`
- AP パスワード: `pass-esp32`

## 4. 認証ロール仕様
[重要] 下記は初期ロール定義である。運用段階では変更可能とし、固定値運用を禁止する。

[新規利用禁止] 下記パスワードを本番固定値として継続利用しない。  
理由: 固定値の長期運用は辞書攻撃・横展開リスクを高めるため。

| ロール | user | password（初期値） | 主な権限 |
|---|---|---|---|
| 閲覧用 | `user` | `esp32user1414` | メイン画面の閲覧のみ |
| メンテナンス者 | `maintenance` | `esp32m1732` | メイン画面、ネットワーク設定変更 |
| 管理者 | `admin` | `esp32adm2718` | メンテナンス者権限 + `k-device` + MQTT証明書設定 |
| メーカー | `mfg` | `esp32m314159` | 管理者権限 + eFuse セキュリティ確認/設定 |

## 5. 画面仕様
### 5.1 メイン画面
- デバイス名
  - 初期値: `IoT_<MACコロン除去・大文字>`
  - 変更可
  - 文字種: 英大/英小/数字/`_`/`-`/`+`
  - 先頭: 英字のみ
  - 最大: 32文字
- サブデバイス名
  - 固定: `IoT_<MACコロン除去・大文字>`
- FWバージョン
- FSデータバージョン
- 接続先情報（読み取り表示）
  - Wi-Fi AP 名
  - MQTT broker（IP / 名前 / Port）
  - Server（IP / 名前 / Port）
  - OTA（IP / 名前 / Port）
  - NTP（IP / 名前 / Port）
- [厳守] 「詳細設定」ボタンは `maintenance` 以上で表示・操作可能とする。

### 5.2 詳細設定画面
[重要] ログインロールにより、表示・編集できる項目を制御する。

- 共通表示:
  - デバイス名、サブデバイス名、FW版、FS版
- `maintenance` 以上:
  - Wi-Fi / MQTT / Server / OTA / NTP の host/name/port/user/password
  - 必要に応じ証明書アップロード
- `admin` 以上:
  - `k-device` 発行番号表示
  - `k-device` 暗号化ファイルアップロード
  - MQTT 証明書設定
- `mfg` のみ:
  - eFuse セキュリティ設定と確認

- 操作ボタン:
  - 設定
  - キャンセル
  - 再起動
  - 設定の確認

[厳守] 「設定の確認」ボタンは「通常モードへ再起動して接続確認（mqtt/ntp/k-device有効性）後、APモードへ再起動して結果表示」を行う。

## 6. API権限制御（画面連動）
- [厳守] 未認証:
  - 設定参照不可
  - 設定更新不可
  - FW更新不可
  - 画像更新不可
- [厳守] `user`:
  - 読み取りのみ（メイン画面）
- [厳守] `maintenance`:
  - ネットワーク設定更新可
- [厳守] `admin`:
  - `maintenance` + `k-device` / MQTT証明書設定可
- [厳守] `mfg`:
  - `admin` + eFuse 操作可
