# IoT ドキュメント概要

[重要] 本書は `IoT` 配下の文書索引です。  
理由: 参照先と更新先を明確化し、設計・実装・試験の整合を維持するため。

## 対象システム
- ESP32-S3（ESP）
- ProductionTool / LocalServer / SecretCore（Windows）
- CloudServer（将来）

## 文書一覧（まず本書を参照）
- `ソース概要.md`  
  [重要] `IoT` 配下のフォルダ責務、変更窓口、`public_id` / AP 認証 / `runPairingSession()` 内部の `createPairingBundle` / 試験関連の参照先を定義する。
- `設計概要.md`  
  目的、全体構成、段階計画（第1〜第4段階）の位置づけを定義。
- `要件定義書.md`  
  機能要件・非機能要件・制約条件を定義。
- `ストレージ仕様書.md`  
  [重要] `NVS` / `LittleFS` の保存区分、`/images` `/certs` `/logs` の責務、画像/証明書更新、ログローテーション方針を定義。
- `WiFi設定更新_詳細設計書.md`  
  Wi-Fi設定更新（AES-GCM暗号、ロールバック、AP復旧、`TPM + wrapped_secret + k-user / k-device` 方式）の詳細設計を定義。
- `鍵管理および初期セットアップ設計仕様書.md`  
  ユーザー独立オンプレ前提の鍵階層、TPM保護、`wrapped_secret`、バックアップ/復元、サポート方針を定義。
- `鍵一覧仕様書.md`  
  `k-user / k-device` 系と `k-iot` 系の鍵一覧、目的、保存場所、実キー生成タイミングを定義する。
- `鍵管理初期セットアップ_実装たたき台.md`

- `機能仕様書.md`  
  段階別の機能仕様（第3段階のみ現行実装対象）を定義。
- `IF仕様書.md`

- `冗長化・多重接続仕様書.md`

## 更新ルール
- [厳守] 仕様変更時は「変更日時」「変更理由」「影響範囲」を必ず記載する。  
  理由: 後続のデグレ防止と判断根拠追跡のため。
- [厳守] セキュリティ関連の変更は `セキュリティ方針_認証情報取扱い.md` を同時更新する。  
  理由: 実装差分と運用ルールの不整合を防ぐため。
- [厳守] 責任範囲や役割分担に関わる変更は `モジュール仕様書.md` を先に更新し、関連文書を追従更新する。  
  理由: 親定義を先に固定しないと、文書ごとに役割解釈がずれるため。
- [厳守] 参照先や変更窓口が増減した場合は `ソース概要.md` と本書を同時更新する。  
  理由: 文書索引とソース索引の不整合を防ぐため。
- [禁止] 認証情報・鍵素材を仕様書例としても記載しない。  
  理由: 文書経由の機密漏えい防止。
