# 通常運用FW・診断用FW差分仕様書

[重要] 本書は量産通常FWと診断用FWの差分方針を定義する。  
理由: `008-0002` の要求である「出力可能ログ種別」「許可する保守コマンド」「切替条件」「顧客環境で恒久運用しない条件」を1か所へ固定し、シリアル方針をぶらさないため。

[厳守] シリアル出力停止は eFuse ではなく FW 制御で実現する。  
理由: `要件定義書.md` `NFR-023` / `NFR-024` / `NFR-025` と整合し、問題解析時に署名済み診断用FWへ一時切替できる余地を残すため。

[禁止] 実行中コマンド受信で通常版と診断版のシリアル状態を切り替える設計を採らない。  
理由: 切替コマンド自体が攻撃面となり、通常版FWへ危険な保守経路を残してしまうため。

## 1. 対象
- 通常運用FW
- 診断用FW
- `ProductionTool` による最終クローズ判断
- `7030` / `7031` による試験確認

## 2. 基本方針
- [厳守] 試験中・不可逆段階確認中は、シリアル観測と復旧余地を残す。
- [厳守] 本番最終クローズでは、シリアル抑止済みの量産通常FWへ更新する。
- [重要] `DIS_USB_SERIAL_JTAG` は必要時のみ書換え不能化として別判断にする。
- [禁止] シリアル停止のみを目的に eFuse を焼かない。
- [推奨] 通常運用FWと診断用FWは版数・ビルド種別・監査ログで明確に識別できるようにする。

## 3. 差分一覧

| 項目 | 通常運用FW | 診断用FW |
|---|---|---|
| 主目的 | 顧客環境での通常運用 | メーカー管理下での一時解析 |
| シリアル出力 | 最小化または停止 | 解析に必要な範囲で有効 |
| 高詳細ログ | [禁止] 常時出力しない | [推奨] 必要時のみ有効 |
| 秘密値出力 | [禁止] 出力しない | [禁止] 出力しない |
| 保守コマンド | [禁止] 危険コマンドを含めない | [重要] 解析用の限定機能のみ許可 |
| 顧客環境常用 | [推奨] 正本 | [禁止] 恒久運用しない |
| 配布条件 | 正規量産版として管理 | メーカー管理下の署名済み配布物に限定 |
| 監査 | 更新理由・版数・適用個体を記録 | 実施者・理由・期間・戻し確認を厳密記録 |

## 3.1 現在のビルド対応 [重要][2026-05-02実装反映]
- `IoT/ESP32/platformio.ini` の `esp32s3_secure_final` を通常運用FWとして扱う。
- `IoT/ESP32/platformio.ini` の `esp32s3_secure` と `esp32s3_secure_rescue` を診断用FW寄りのビルドとして扱う。
- [厳守] 通常運用FWでは `APP_ENABLE_DIAGNOSTIC_LOG=0`、`APP_ENABLE_FACTORY_APIS=0` とする。
- [厳守] 診断用FWでは `APP_ENABLE_DIAGNOSTIC_LOG=1`、`APP_ENABLE_FACTORY_APIS=1` とする。
- [重要] 実装上の切替点は `IoT/ESP32/header/firmwareMode.h`、`IoT/ESP32/src/log.cpp`、`IoT/ESP32/src/maintenanceApServer.cpp` とする。

## 4. 通常運用FWの要件
- [厳守] 通常運用に不要な高詳細ログを停止または最小化する。
- [厳守] シリアル経由の危険な保守コマンドを含めない。
- [厳守] パスワード、鍵、トークン、平文認証情報などの秘密値を出力しない。
- [推奨] 障害判定に必要な最小限のエラーコードと状態要約のみを残す。
- [推奨] ファイルログはシリアル出力と別経路で保存し、必要時のみ有効化する。
- [実装反映][2026-05-02] 既定シリアルログレベルは `WARN` とし、`pairing` / `production` 系AP APIは `403 factory api disabled in normal firmware` で拒否する。

## 5. 診断用FWの要件
- [厳守] 診断用FWはメーカー管理下の署名済み配布物としてのみ扱う。
- [厳守] 使用目的は障害解析・原因切り分け・復旧確認に限定する。
- [厳守] 使用後は通常運用FWへ戻し、恒久運用しない。
- [禁止] 顧客環境で診断用FWを常用しない。
- [推奨] 診断用FWで有効化するログ種別と保守機能は最小限に限定し、都度監査記録を残す。
- [実装反映][2026-05-02] 診断用FW寄りビルドでは `DEBUG/INFO` ログと `pairing` / `production` 系AP APIを有効化し、解析・製造作業のみに使う。

## 6. 切替条件
- [厳守] 通常運用FW -> 診断用FW の切替は、問題解析が必要であると判断された場合に限る。
- [厳守] 診断用FW -> 通常運用FW の戻しは、解析完了後に必ず実施する。
- [重要] 本番最終クローズでは「シリアルを止めたバージョンとしての通常運用FW」へ更新することを正とする。
- [重要] `ProductionTool` は最終クローズ判断と証跡記録を担うが、FW 切替自体は量産通常FWの更新手順で実施する。

## 7. `ProductionTool` との関係
- [厳守] `ProductionTool` の段階4では、`ENABLE_SECURITY_DOWNLOAD` を最後に扱う。
- [重要] `PRODUCTION_TOOL_BURN_DIS_USB_SERIAL_JTAG=1` は必要時のみ使用し、書換え不能化の判断として扱う。
- [重要] シリアル出力停止は `ProductionTool` の eFuse 自動焼成ではなく、最終クローズ後の通常運用FW更新で実現する。
- [推奨] `runner-precheck-summary.json` に最終クローズで通常運用FW更新が必要であることを明示し、作業者が途中で誤解しないようにする。

## 8. 試験との対応
- `7030`: 通常運用FWのシリアル出力制約試験
- `7031`: 診断用FWの一時シリアル有効化試験
- `7087`: `ProductionTool` での最終クローズ判断・証跡確認

[厳守] `7030` / `7031` を実施し、通常版と診断版の差分が試験記録へ残っていない限り、本仕様の運用完了とみなさない。
