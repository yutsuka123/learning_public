# ProductionTool画面仕様書

[重要] 本書は `ProductionTool` の最小画面構成、画面遷移、表示項目、監査ログ出力先を定義する。  
理由: `LocalServer` と分離した工場内専用ツールとして、基本機能を先に固定し、不可逆処理本体の実装前でも UI 導線と安全停止条件をブレなくするため。

[厳守] `ProductionTool` の実装フォルダ名は `IoT/ProductionTool/` とし、`ProductionTool` / `ProductionTool` の名称を新規利用しない。  
理由: フォルダ名、ソフト名、文書名の不一致をなくすため。

[厳守] 本書の対象は `004-0008`〜`004-0010` の基本機能安定化フェーズであり、eFuse / Secure Boot / Flash Encryption の不可逆処理本体はまだ実行対象にしない。  
理由: 先に起動、追加認証、対象機確認、dry-run、監査ログ導線を安定させる必要があるため。

[禁止] `LocalServer` の通常運用画面へ `ProductionTool` の画面やボタンを混在表示しない。  
理由: 顧客向け通常運用と工場内高リスク操作の境界を画面レベルで明確に分離するため。

## 1. 対象範囲
- `ProductionTool` 単独起動時の最小画面構成
- 追加認証
- 対象機確認
- 実行内容確認
- dry-run
- 監査ログ表示導線
- 失敗時の安全停止

## 2. 画面一覧
- `PT-001` 起動画面
- `PT-002` 追加認証画面
- `PT-003` 対象機選択・識別確認画面
- `PT-004` 実行内容確認画面
- `PT-005` dry-run 実行・結果画面
- `PT-006` 監査ログ確認画面
- `PT-007` エラー / 安全停止画面

## 3. 画面仕様
### 3.1 `PT-001` 起動画面
- 目的:
  - アプリ起動直後に `ProductionTool` 専用画面であること、版数、実行モード、接続初期化状態を明示する。
- 表示項目:
  - アプリ名
  - 版数
  - 実行環境（工場端末名、OS、起動日時）
  - `SecretCore` 共通部接続状態
  - 監査ログ出力先
  - 「開始」ボタン
  - 「終了」ボタン
- [重要] 接続初期化が未完了でも画面は表示するが、次画面へは進ませない。
- [厳守] `SecretCore` 共通部接続失敗時は `PT-007` へ遷移する。

### 3.2 `PT-002` 追加認証画面
- 目的:
  - 通常管理者とは別のメーカーモード専用認証で `ProductionTool` 操作を保護する。
- 入力項目:
  - 操作者 ID（任意。空欄可）
  - メーカーモード専用パスワード
  - 作業指示番号（任意。空欄時は `YYYYMMDDHHMMSS` を自動採番）
- ボタン:
  - 「認証」
  - 「戻る」
  - 「終了」
- [厳守] 認証失敗時は対象機画面へ遷移させない。
- [厳守] 平文パスワードをログへ出力しない。
- [推奨] 連続失敗回数、操作者 ID（空欄時は `<empty>`）、端末名、時刻を監査ログへ残す。
- [厳守] 通常管理者パスワードを `PT-002` の認証へ流用しない（メーカーモード専用パスワードのみ許可）。
- [厳守] 開発用固定値（`maker2026`）は暫定運用とし、本番運用では以下を満たす。
  - 工場セキュリティ管理責任者/副責任者を定義し、配布責任を明確化する。
  - 90日以内の定期更新、または異動/漏えい疑い時の即時更新を行う。
  - 旧値失効後は新値のみ有効にし、旧値認証試行は監査ログへ記録する。
  - 認証情報の実値はリポジトリ、仕様書、標準出力に記載しない。
- [重要] `004-0010` 再判定までに本番運用値と更新/失効手順を確定し、`005-0003` を完了扱いにする。

### 3.3 `PT-003` 対象機選択・識別確認画面
- 目的:
  - 対象機一覧からの行選択で取り違えを防ぎ、誤操作の入口を減らす。
- 表示項目:
  - 対象機一覧（デバイスごとに1行）
  - シリアル番号
  - MAC アドレス
  - `public_id`
  - FW 版数
  - セキュア化状態
  - 接続状態（`MQTT接続中` / `APモード`）
- 入力項目:
  - 一覧の選択行番号
- ボタン:
  - 「選択確定」
  - 「再読込」
  - 「戻る」
  - 「終了」
- [厳守] 未選択または範囲外行の選択時は次画面へ進ませない。
- [禁止] この画面で不可逆処理を直接開始できるボタンを置かない。
- [重要] 監査ログには、`selectedIndex`、`serial`、`mac`、`public_id`、`connectionStatus`、`selectionResult` を保存する（秘密値は保存しない）。

### 3.4 `PT-004` 実行内容確認画面
- 目的:
  - 実行予定内容、前提条件、危険性、dry-run で確認する範囲を操作者へ明示する。
- 表示項目:
  - 実行予定工程一覧
  - 事前条件チェック結果
  - 実行禁止条件
  - 出力ログ保存先
  - dry-run 対象範囲
- ボタン:
  - 「dry-run 実行」
  - 「戻る」
  - 「終了」
- [厳守] この段階では最終実行ボタンを表示しない。
- [重要] 前提条件不足時は `PT-007` へ遷移する。

### 3.5 `PT-005` dry-run 実行・結果画面
- 目的:
  - 不可逆処理本体へ入らずに、導線、認証、対象機確認、監査ログ出力、停止点を確認する。
- 表示項目:
  - dry-run 開始時刻
  - 実行ステップ一覧
  - 現在ステップ
  - 成功 / 失敗 / 中断
  - 停止理由
  - 監査ログファイル名
- ボタン:
  - 「結果保存」
  - 「監査ログ確認」
  - 「起動画面へ戻る」
  - 「終了」
- [厳守] dry-run は eFuse / Secure Boot / Flash Encryption の本実行手前で必ず停止する。
- [禁止] dry-run 成功をもって不可逆処理本体完了とみなさない。

### 3.6 `PT-006` 監査ログ確認画面
- 目的:
  - 基本機能試験時に、起動、認証、対象機確認、dry-run、エラーの証跡を即座に確認できるようにする。
- 表示項目:
  - ログファイル一覧
  - 最新ログの概要
  - イベント種別
  - 実施者 ID
  - 対象機識別子
  - 結果
  - 時刻
- ボタン:
  - 「再読込」
  - 「ファイルを開く」
  - 「戻る」
- [厳守] 監査ログ画面でも秘密値は伏字または fingerprint のみとする。

### 3.7 `PT-007` エラー / 安全停止画面
- 目的:
  - 基本機能段階で異常が出た場合に、不可逆処理へ進まず停止する。
- 表示項目:
  - エラーコード
  - エラー概要
  - 失敗ステップ
  - 推奨対応
  - 監査ログ参照先
- ボタン:
  - 「ログ確認」
  - 「起動画面へ戻る」
  - 「終了」
- [厳守] 失敗時は自動で `PT-005` 以降の続行を行わない。

## 5. 監査ログ仕様
- [重要] 基本機能安定化フェーズでは、少なくとも以下を監査ログ対象とする。
  - 起動
  - 終了
  - 追加認証成功 / 失敗
  - 対象機選択
  - 対象機確認不一致
  - dry-run 開始
  - dry-run 正常終了
  - dry-run 異常終了
  - 安全停止
- ログ出力先:
  - 既定候補: `%PROGRAMDATA%\\IoT\\ProductionTool\\logs\\audit\\`
  - [将来対応] 正式なインストール先決定後に `003-0016` / `009-1021` と合わせて確定する。
- ログファイル名:
  - `audit-YYYYMMDD-HHMMSS.json`
- [厳守] パスワード、raw key、中間秘密、eFuse 実値は保存しない。
- [推奨] 端末名、操作者 ID、対象機識別子、結果、エラーコード、時刻、実行フェーズを保存する。

## 6. `LocalServer` / `ProductionTool` 分離ルール
- [厳守] `LocalServer` から `ProductionTool` の起動ボタンや eFuse 実行導線を直接提供しない。
- [厳守] `ProductionTool` は `LocalServer` と別の実行物、別のインストーラ、別の監査ログ保存先を持つ。
- [重要] `SecretCore` 共通部は再利用してよいが、画面、配布物、認証入口は分離する。
- [厳守] 上記の共通化は `LocalServer` 側 `SecretCore` 共通部の再利用を意味し、`ProductionTool` を `LocalServer` と一体化する意味ではない。

## 9. 後続フェーズ secure flow 設計（005-0006 / 005-0007）

[重要] 本節は `ProductionTool` 基本機能安定化フェーズの次段として、不可逆処理本体へ入る前の専用フローと事前チェック責務を固定する。  
理由: `runProductionSecureFlow()`、ESP32 側専用処理、監査ログ、隔離判定の責務境界を先に定義し、通常OTA/通常MQTT/AP保守導線との混線を防ぐため。

- [将来対応][2026-04-23] `008-0041` / `008-0043` では GUI 化を進めるが、本節の工程ロック、対象機再確認、dry-run、安全停止、4点証跡条件はそのまま継承する。`008-0042` は最終運用シリアル方針として退避済みのため、GUI 化の参照先から除外する。

### 9.1 ProductionTool 専用セキュア化フロー
- [厳守] 不可逆処理は `SecretCore` の `runProductionSecureFlow()` からのみ開始する。
- [禁止] 通常 OTA、通常アプリ、通常 MQTT 指令、`LocalServer` 管理画面から eFuse / Secure Boot / Flash Encryption の本実行を開始しない。
- [厳守] ESP32 側には ProductionTool 専用のセキュア化ステートマシンを設け、通常運用タスクと責務を分離する。
- [重要] 想定ステートは `precheck` -> `lock_environment` -> `stage_prepare` -> `stage_execute` -> `stage_readback_verify` -> `final_judgement` とする。
- [厳守] `stage_execute` では 1 ステップごとに対象 eFuse / 設定項目を限定し、各ステップ直後に読戻し検証へ進む。
- [厳守] 読戻し検証 NG 時は次ステップへ進まず、その時点の読戻し結果と監査ログを根拠に `final_judgement=isolated` または `failed` とする。
- [厳守] 「途中まで焼けたら残りも続行する」前提を採用しない。
- [重要] `PT-004` はこの secure flow の実行予定段階と事前チェック結果を表示する画面とし、`PT-005` dry-run は `stage_execute` 手前で停止する。
- [重要] 不可逆処理本体を将来 GUI 化する場合でも、`PT-001`〜`PT-007` の前段導線を維持したうえで後続画面として追加する。

- [重要][2026-04-25] 経路が複数ある場合は、`ProductionTool` / `AP` / `LocalServer` などの該当経路へそれぞれ同内容を通知し、監査ログで経路ごとの伝達結果を判別できるようにする。

### 9.4 実行完了後の自動検証設計（005-0008 観点確定）[2026-04-28]
[重要] 本節は `006-0011` 試験記録（`試験記録書.md` の workflow 解禁、`7041` / `7083` / `espefuse` 証跡）を前提に、**不可逆段階の実行直後〜次段階直前**に必須となる自動（または半自動）検証の観点を固定する。  
理由: `007-0004` / `007-0006` の 4点証跡と手順を、`ProductionTool` 利用者が参照できる単一の表に落とし込み、`005-0008` の「設計」を完了可能な粒度にするため。

#### 9.4.1 責務分界（現行実装との対応）
- [厳守] `SecretCore` の `runProductionSecureFlow()` は、`dryRun=false` かつ `allowIrreversibleExecution=true` のとき **`stage_execute` 受理**と **`stage_readback_verify` / `final_judgement` 相当の状態遷移**を記録しうるが、**eFuse の実書込み本体とデバイス上の真偽検証は ProductionTool 側オーケストレーション**に委ねる（`generic_workflow.rs` の設計コメントと整合）。
- [重要] したがって **`state=completed result=OK` は「不可逆実行パスの解禁・受理が整合した」証跡**であり、単体では **Secure Boot / Flash Encryption の実機有効**や **OTA 継続**の証明にはならない。2026-04-27 の `006-0011` 記録（解禁前は `final_judgement result=blocked`）と同じ読み方をすること。

#### 9.4.2 検証観点一覧（必須）
