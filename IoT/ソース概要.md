# IoT ソース概要

[重要] 本書は `IoT` 配下のソース/フォルダ索引です。  
理由: 変更対象のフォルダと参照すべき仕様書を迷わず辿れるようにし、実装・文書・試験の整合を保つため。

## 1. 参照順序
- `ドキュメント概要.md`: まず確認する文書索引。
- `ソース概要.md`: どのフォルダ/ソースを変更するか判断する索引。
- `モジュール仕様書.md`: [重要] 責任範囲と役割境界の親定義。
- `要件定義書.md`: 変更の要件境界を確認する。
- `機能仕様書.md` / `IF仕様書.md`: 振る舞いと IF を確認する。
- `ストレージ仕様書.md`: `NVS` / `LittleFS` の保存区分、`/images` `/certs` `/logs`、画像/証明書更新、ログ保持を確認する。
- `冗長化・多重接続仕様書.md`: `serverId` / `brokerId` / `wifiApId`、多重接続、冗長化、`k-user` / `k-device` の運用前提を確認する。
- `セキュリティ方針_認証情報取扱い.md`: セキュリティ制約を確認する。
- `試験仕様書.md` / `試験記録書.md`: 試験観点と証跡更新先を確認する。

## 2. フォルダ概要
- `ESP32/`
  ESP32-S3 実装本体。MQTT / HTTPS / OTA / AP モード / NVS 保存 / LittleFS 更新 / ログ保存を担当する。
- `LocalServer/`
  TypeScript / Node.js ベースのローカルサーバー。Web UI、REST API、MQTT 連携、OTA 配布、運用状態管理を担当する。
- `ProductionTool/`
  [重要][2026-03-17] 工場端末向け `ProductionTool` の実装格納先。`LocalServer` と分離した起動物、追加認証、対象機確認、dry-run、監査ログ導線の実装入口。
- `LocalSoft/`
  [廃止の方針][念のため保存] 旧ローカル運用ソフト。新規実装は `LocalServer/` 側へ統合する。
- `CloudServer/`
  [将来対応] クラウド側実装格納先。現時点では主対象外。
- `shared/`
  ESP32 / Local / Cloud 間で共有する公開定義格納先。topic 名、共通定数、protocol version などを置く。

## 4. テーマ別の参照先
### 4.1 `public_id`
- 仕様確認: `要件定義書.md`、`機能仕様書.md`、`IF仕様書.md`
- MQTT 反映: `MQTTコマンド仕様書.md`、`shared/include/common.h`
- [仕様変更] 初期値は `IoT_<macアドレスからコロン除去>` とする。

### 4.2 AP 共通トップ画面 / ProductionTool 追加認証
- 要件/機能: `要件定義書.md`、`機能仕様書.md`
- 認証ルール: `セキュリティ方針_認証情報取扱い.md`
- 操作確認: `コマンド仕様書.md`、`試験仕様書.md`
- `ProductionTool` 基本導線: `ProductionTool画面仕様書.md`、`モジュール仕様書.md`、`IF仕様書.md`
- [厳守] `ProductionTool` は `LocalServer` と別ソフト、別実行物として独立動作させる。

### 4.3 `runPairingSession()` / `createPairingBundle` / 初回投入 / 再ペアリング
- IF / 内部項目: `IF仕様書.md`
- 鍵の役割: `鍵一覧仕様書.md`
- 秘密処理境界: `LocalServer秘密処理別層仕様書.md`
- 試験/手順: `試験仕様書.md`、`コマンド仕様書.md`
- [厳守] 外部公開 API は `/api/workflows/...` 系に限定し、`createPairingBundle` は `runPairingSession()` 内部 helper とする。
- [厳守] TS へ raw `k-device` や `k-pairing-session` を返さない。

### 4.4 `k-user` 再発行と current/previous 運用
- 機能仕様: `機能仕様書.md`
- セキュリティ/監査: `セキュリティ方針_認証情報取扱い.md`
- 試験観点: `試験仕様書.md`、`試験記録書.md`
- [重要] 稼働時間 48h で旧鍵失効、240h 保持後に previous key 自動削除とする。

### 4.5 ストレージ更新
- 保存方針: `ストレージ仕様書.md`
- MQTT IF: `MQTTコマンド仕様書.md`
- 実装責務: `モジュール仕様書.md`
- 試験観点: `試験仕様書.md`
- [厳守] 鍵情報全般、Wi-Fi / MQTT / HTTP / HTTPS / NTP / 認証情報は `NVS` を正とする。
- [厳守] 画像、証明書、ログは `LittleFS` の `/images` `/certs` `/logs` に分離する。
- [厳守] 画像・証明書更新は差分更新とし、`SHA-256` 一致時のみ反映する。

### 4.6 多重接続 / 冗長化
- 親仕様: `冗長化・多重接続仕様書.md`
- 鍵/保護方式: `鍵一覧仕様書.md`、`セキュリティ方針_認証情報取扱い.md`
- MQTT IF: `IF仕様書.md`、`MQTTコマンド仕様書.md`
- [厳守] `Server` と `MQTT Broker` は別責務として扱う。
- [厳守] `ESP32` は将来の複数 `Server` 同時接続を考慮した識別子と保存形式で設計する。
- [厳守] `WiFi AP` は `ESP32` ごとに固定1件としつつ、システム全体では複数存在してよい。

## 5. 変更時のルール
- [厳守] ソース変更時は、関連する仕様書と本書の索引記載を同時確認する。
- [厳守] 責任範囲に関わる判断は `モジュール仕様書.md` を親定義として扱う。
- [厳守] MQTT topic / payload を変更した場合は `MQTTコマンド仕様書.md` を同時更新する。
- [厳守] AP 認証、ProductionTool 認証、pairing、鍵保存方法を変更した場合は `セキュリティ方針_認証情報取扱い.md`、`IF仕様書.md`、`試験仕様書.md` を同時更新する。
- [禁止] 秘密値、実パスワード、秘密鍵、実証明書、raw `k-device` を文書例へ記載しない。
