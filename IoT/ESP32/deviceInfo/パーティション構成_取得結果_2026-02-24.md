# パーティション構成 取得結果（2026-02-24）

[重要] 本書は接続中デバイス（COM9）から実際に読み出したパーティション構成の記録です。  
理由: 参照CSVの根拠を残し、将来の構成変更差分を追跡可能にするため。

## 1. 取得条件
- デバイス: ESP32-S3
- COMポート: COM9
- 読出し元: Flash `0x8000` から `0x1000` バイト
- 使用コマンド:
  - `python -m esptool --port COM9 read_flash 0x8000 0x1000 IoT/ESP32/deviceInfo/partition_table_0x8000_0x1000.bin`

## 2. 取得結果（CSV化）
- 保存CSV:
  - `IoT/ESP32/partitions/esp32s3_from_device_16MB_COM9.csv`

## 3. パーティション一覧
- nvs: `0x9000` / `0x5000`
- otadata: `0xe000` / `0x2000`
- app0: `0x10000` / `0x140000`
- app1: `0x150000` / `0x140000`
- spiffs: `0x290000` / `0x160000`
- coredump: `0x3f0000` / `0x010000`

## 4. 合計サイズ確認
- 最終領域終端: `0x3f0000 + 0x010000 = 0x400000`
- 実効使用領域: 4MBレンジ内構成
- 補足: デバイスFlash自体は16MB検出。未使用領域は将来拡張可能。

## 5. 運用方針
- [厳守] セキュリティ設定変更時（暗号化/セキュアブート適用時）は再取得して履歴化する。
- [禁止] 根拠なしにパーティションCSVだけ変更しない。
