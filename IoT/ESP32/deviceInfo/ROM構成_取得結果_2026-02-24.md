# ESP32-S3 ROMアドレス/構成 取得結果（2026-02-24）

[重要] 本書はCOM接続実機（COM9）に対して実施したROMアドレス調査結果です。  
理由: 第3段階のセキュアブート/暗号化導入前に、メモリ観点の現状証跡を残すため。

## 1. 取得対象
- 対象デバイス: ESP32-S3 (rev v0.2)
- 接続ポート: COM9
- 取得日: 2026-02-24

## 2. 実施コマンド
- `python -m esptool --port COM9 read_mem <address>`
- `python -m esptool --port COM9 dump_mem 0x40000000 0x00000400 IoT/ESP32/deviceInfo/rom_dump_0x40000000_0x400.bin`

## 3. ROMアドレス調査（read_mem）
- `0x40000000 = 0x1049c500`
- `0x40010000 = 0x0008e020`
- `0x40020000 = 0x607198d7`
- `0x40030000 = 0x85500a0c`
- `0x40070000 = 0x00000000`

補足:
- `0x40000000` 近辺は命令ROM領域として応答があり、ROM空間である可能性が高い。
- `0xbad00bad` 応答が返るアドレスは、当該タイミングで有効マップされていない領域と判断できる。

## 4. 参照アドレス試験結果（構成確認用）
- `0x3C000000 = 0xbad00bad`
- `0x3C800000 = 0xbad00bad`
- `0x3D000000 = 0xbad00bad`
- `0x3FC80000 = 0x00000000`
- `0x3FCE0000 = 0x4ad5abbe`
- `0x60000000 = 0x000000ff`

## 5. ROMダンプ結果
- [念のため保存] 生ROMダンプファイルは取得済みだったが、運用方針により削除済み。
- 削除理由: 今後知っていても仕方のない生バイナリ情報であり、不要データ管理を避けるため。

## 6. ROM構成について
- [重要] ROMの厳密な全体構成（全セクション境界、用途名）は、実機read_memだけでは完全確定できない。
- [推奨] ESP32-S3 Technical Reference Manual / ESP-IDFメモリマップ仕様と併読して、最終構成表を確定する。
- [厳守] セキュリティ設定有効化前後で同手順を再実施し、差分を保存する。
