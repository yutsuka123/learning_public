# IoT 試験仕様書

[重要] 本書は LocalServer 初期実装の疎通・機能試験仕様を定義する。  
理由: 実装後の合否判定を再現可能にするため。

## 1. 試験方針
- 試験番号は4桁（上1桁: 分類、下3桁: 連番）で管理する。
- 7000番台をネットワーク/サーバー系試験として利用する。
- [厳守] 実施結果は `試験記録書.md` に1対1で記録する。

## 2. 試験項目
- 7001: LocalServer 起動試験
  - 手順: `cd IoT/LocalServer && npm install && npm run build && npm run start`
  - 期待結果: `HTTP started` が出力され、`/api/health` が `status=ok` を返す。
- 7002: MQTT 接続試験
  - 手順: `cd IoT/LocalServer && npm run test:connect`
  - 期待結果: MQTTへ接続し `esp32lab/call/status/all` publish成功を出力する。
- 7003: Web GUI 一覧表示試験
  - 手順: `http://localhost:3100/` を開き、`/api/devices` 応答確認。
  - 期待結果: テーブル描画ができ、操作ボタンが押下可能。
- 7004: statusコマンド送信試験
  - 手順: `POST /api/commands/status` に `{"targetNames":"all"}` を送信。
  - 期待結果: `result=OK` が返り、MQTT publishされる。
- 7005: OTAコマンド送信試験
  - 手順: `POST /api/commands/ota` に `{"targetNames":"all"}` を送信。
  - 期待結果: `result=OK` が返り、`otaStart` publishされる。
- 7006: OTA配布エンドポイント試験
  - 手順: `/ota/manifest.json` と `/ota/firmware.bin` を取得。
  - 期待結果: manifest取得成功。firmware未配置時は404を返す。
- 7007: OTA進捗通知試験（デバイス別）
  - 手順: OTA実行中に `esp32lab/notice/otaProgress/+` を購読し、LocalServer画面を確認。
  - 期待結果: デバイス別に進捗率（例: 30%）が表示される。
- 7008: OTA同一進捗率リトライ試験
  - 手順: 通信を劣化させ、同一進捗率で停止する状況を作る。
  - 期待結果: 5秒間隔で最大3回まで同一進捗率リトライを行う。
- 7009: OTA最初から再試行試験
  - 手順: 7008で回復不可条件を維持する。
  - 期待結果: 最初から再試行を最大3回実施し、上限超過でNG通知する。
- 7010: SHA256一致試験
  - 手順: 正常な `firmware.bin` と一致するSHA256を配布指令へ設定する。
  - 期待結果: SHA256一致後のみ更新確定し、次回起動先が更新領域へ切替わる。
- 7011: SHA256不一致試験（異常系）
  - 手順: 意図的に誤SHA256を `otaStart` に設定する。
  - 期待結果: `shaMismatch` としてNG終了し、更新確定しない。
- 8001: 異常系（MQTT認証失敗）
  - 手順: `.env` の `MQTT_PASSWORD` を誤値にして起動。
  - 期待結果: 接続失敗ログが出力され、プロセスが異常終了しない。
