# IoT 環境仕様書

[重要] 本書は開発・検証環境を固定化し、再現可能な構築を可能にする。  
理由: セキュリティ有効時は環境差異による不具合切り分けが難しいため。

## 1. 対象環境
- Local OS: 現行は Windows（TPM 2.0 必須）
- [将来対応] macOS は同等のハードウェア拘束方式確定後に対応
- ESP: ESP32-S3
- MQTTブローカ: mosquitto（Local PC上、TLS + ID/パスワード認証必須）

## 1.1 環境責任範囲
- [厳守] `LocalServer` は通常運用環境、`SecretCore` はローカルIPC限定の秘密処理環境、`ProductionTool` は工場内専用環境として分離する。
- [厳守] `SecretCore` は TCP/HTTP 待受を持たず、Named Pipe / Unix Domain Socket 等のローカルIPCのみを利用する。
- [厳守] `ProductionTool` 用の実行環境、認証情報、操作端末を通常運用の `LocalServer` 環境へ混在させない。
- [厳守] Rust 共通化は `LocalServer` 側 `SecretCore` 共通部の再利用を意味し、`ProductionTool` の実行環境を `LocalServer` へ統合する意味ではない。
- [厳守] 詳細な責任境界は `モジュール仕様書.md` を親定義とする。

## 2. 必須ツール
- PlatformIO（ESPビルド）
- Python（PlatformIO依存）
- mosquitto / mosquitto-clients
- .NET SDK（LocalをC#で実装する場合）
- Pythonランタイム（Local/CloudをPythonで実装する場合）

## 3. PlatformIO方針
- [重要] 実行用の PlatformIO 設定は `IoT/ESP32/platformio.ini` を正とする。`IoT/platformio.ini.example` はビルド入口の案内のみであり、旧ひな形との混同を避ける。
- [厳守] `platformio.ini` でプラットフォーム、フレームワーク、ライブラリをバージョン固定する。
- [厳守] ボードはESP32-S3系を指定する。
- [推奨] ビルド環境は `env` を分割し、開発・検証で設定を分離する。
- [推奨] ESPのC++コードは `-std=gnu++20` 等でC++20利用を優先する。
- [厳守] 7025 の `未確定起動失敗` 試験を行う環境では `IoT/ESP32/sdkconfig.defaults` の `CONFIG_BOOTLOADER_APP_ROLLBACK_ENABLE=y` を有効にする。

## 3.1 安全消去ポリシー（003-0018）
- [重要] アンインストール・廃止時における機密データの消去要件は `安全消去ポリシー.md` を参照する。
- [厳守] 通常削除のみで機密残留を許容する運用は禁止する。復元困難化（上書き消去またはOS安全削除API）を採用する。
- 理由: `LocalServer` / `ProductionTool` / `LocalSoft` 廃止時の機密漏えいを防止するため。
- LocalServer インストール/アンインストールの具体的な手順は `コマンド仕様書.md` 4.1 を参照する。理由: 再現可能な運用のため。

## 4. セキュリティ環境要件（第3段階）
- ROM暗号化有効前後で起動確認手順を分離する。
- セキュアブート鍵素材はローカル安全領域に保存し、リポジトリへ保存しない。
- eFuse適用手順は実行前チェックリストを用意する。
- [厳守] mosquittoはTLS有効設定（CA/サーバー証明書）を適用する。
- [厳守] mosquittoはログインID/パスワード認証を有効化する。
- [厳守] `LocalServer` の OTA配布ポート `4443/TCP` は Windows Firewall で受信許可する。  
  理由: ESP32 から `https://mqtt.esplab.home.arpa:4443/ota/firmware.bin` へ到達できないと OTA本体が開始できないため。
- [禁止] 検証回避のためにTLS検証を無効化した設定を常用しない。
- [厳守] `wrapped_secret` または暗号化済みバックアップファイルで `S_random` / `k-user` を保護し、平文保存を禁止する。
- [推奨] Wi-Fi設定更新（暗号化/ロールバック/AP復旧）の試験環境を分離する。
- [厳守] APメンテナンスモードの認証情報は `IoT/LocalServer/.env`、`IoT/ESP32/.env`、`IoT/ESP32/header/sensitiveData.h` で管理し、ESP32側の `.env` と `sensitiveData.h` は同一情報を維持する。
- [厳守] 共有用サンプルは `IoT/LocalServer/.env.example.sample.txt` と `IoT/ESP32/.env.example.sample.txt` を使用し、ダミー値のみ記載する。
- [重要][2026-03-25] フェーズGの製品版セキュリティ方針（`production-hmac-v1`、`AES-128`、`BLOCK_KEY0/1/2`、`nvs`、`ENABLE_SECURITY_DOWNLOAD`）は `IoT/ESP32/.env`、`IoT/ESP32/.env.example.sample.txt`、`IoT/ESP32/header/sensitiveData.h` に同時反映し、1台目/2台目で同一方針を維持する。
- [厳守][2026-03-25] 上記3ファイルへ記載するのは**非秘密な設定方針のみ**とし、`fe_key.bin` / `sb_signing_key.pem` / `hmac_nvs_key.bin` の実値や保存先実パスは記載しない。
