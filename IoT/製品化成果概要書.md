# IoT 製品化成果概要書

[重要] 本書は、今回の製品レベル化に向けて「何ができたか」「どのモジュールで何を担当しているか」「何の技術で動いているか」を要約する。  
理由: 実装成果と残課題を同じ視点で把握し、次工程（量産展開、GUI化、NVS改善）へ引き継ぎやすくするため。

[厳守] 成果は「実装済み」と「未完了/将来対応」を分けて記載する。  
理由: 完了済みと検討中を混同すると、出荷判断を誤るため。

[禁止] 実鍵、認証情報実値、端末依存の秘密設定を記載しない。  
理由: 成果説明資料から機密が漏れるのを防ぐため。

## 1. 今回できたこと
- `ESP32-S3` 実機で `Flash Encryption`、`NVS(HMAC鍵投入)`、`Secure Boot v2`、`JTAG封鎖 + Secure Download` の段階適用を実施した。
- `MQTT over TLS`、`OTA`、APモード保守、通常運用を、最終不可逆工程後も継続確認できる状態まで持っていった。
- `ProductionTool`、`LocalServer`、`SecretCore` の役割分離を保ったまま、製造系と通常運用系を混在させない構成を維持した。
- `7100` により「シリアルログは見えるが、未認証の読出し・危険操作は実行できない」ことを、設備なし条件でも非破壊で確認できる手順へ整理した。

## 3. 製品レベルに近づいた安全要素
- `Flash Encryption`: 実機で有効化済み。
- `Secure Boot v2`: 実機で有効化済み。署名済みバイナリ起動を確認済み。
- `eFuse` 封鎖: `DIS_PAD_JTAG`、`DIS_USB_JTAG`、`ENABLE_SECURITY_DOWNLOAD` を適用済み。
- `JTAG/読出し抑止`: `get_security_info`、`read_mem` 拒否、`read_flash` 拒否で非破壊確認済み。
- `MQTT暗号化`: `TLS + ID/パスワード` 前提で運用。
- OTA継続性: 最終不可逆工程後も signed OTA workflow と再オンライン復帰を確認済み。

## 4. 現時点での制約と残課題
- [重要] `NVS` は設計意図の `patternC` ではなく、現状は PlatformIO/Arduino 制約下で `patternA` 動作である。
- [重要] `patternA` でも `Flash Encryption` により物理読出し耐性は確保しているが、「NVS初期化処理としての理想形」は `patternC` である。
- [重要] `7099` の最終整理は次セッションへ持ち越しであり、通常/APの最終観測の扱いをレビューゲート前に再確認する必要がある。
- [重要] anti-rollback、`NVS patternA/patternC` 両立ロジック、`ProductionTool` GUI化は `008` 以降の継続課題である。

## 5. `patternA` と `patternC` の整理
- `patternA`: アプリ実行時に `CONFIG_NVS_ENCRYPTION` が見えず、平文NVS初期化経路を通る実装整理。現状はこの経路で安定動作している。
- `patternC`: 設計意図の secure NVS 初期化経路。`HMAC` 前提の安全な初期化ロジックを通す。
- [重要] 今後の目標は「開発中に詰まらない fallback」と「製品版での secure path」を両立することであり、`patternA` 固定を最終解としない。
- [合意済み][2026-03-27] 開発時は secure を先に試し、secure 条件不足時のみ plain fallback を許可する。fallback 理由はログへ必ず残す。
- [合意済み][2026-03-27] 製品時は secure 条件が揃っているなら secure 固定とし、plain fallback は許可しない。secure 初期化失敗時は `NG` 扱いで停止または隔離判定とする。
